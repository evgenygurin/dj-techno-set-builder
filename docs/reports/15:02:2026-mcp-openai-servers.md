# Отчёт: MCP/OpenAI — базовый стек серверов

**Дата:** 2026-02-15  
**Контекст:** репозиторий уже содержит доменный MCP-сервер (`dj-techno`) на FastMCP; добавлена документация, как мыслить “контекстом” и какие классы MCP-серверов полезно подключать рядом для работы через OpenAI-клиент.

## Summary

- В `docs/data-inventory.md` добавлен раздел **«MCP / OpenAI: контекст и рекомендованные серверы»**:
  - что уже есть (gateway, транспорты, точки входа)
  - рекомендованный “базовый стек” MCP-серверов рядом с `dj-techno` (FS/Git/DB/Fetch/Browser)
  - минимальные правила безопасности (least-privilege, allowlist, redaction)

## Decisions

- Не перечислялись конкретные реализации/пакеты серверов, а зафиксированы **классы** MCP-серверов и принципы безопасного подключения — чтобы документ оставался стабильным и не ломался из‑за смены инструментов.
- Добавлено именно в **data-inventory**, чтобы связать “какие данные есть” ↔ “как LLM к ним получает доступ”.

## Risks

- Если нужно “под ключ” (конкретные команды/пакеты/конфиги под конкретного OpenAI-клиента), потребуется зафиксировать целевой клиент и предпочитаемые реализации MCP-серверов.

## Blockers

- Нет.

## Next step

- При необходимости: добавить готовый пример конфигурации для выбранного OpenAI-клиента (с точными именами серверов, транспортом и allowlist’ами).
